SQL Queries with Demo

1. INSERT - Add New Auction Item (CREATE)

INSERT INTO auction_items (item_name, seller_id, category, description, starting_price, end_date, status) 
VALUES ('Vintage Rolex Watch', 1, 'Antiques', 'Rare 1965 Rolex Submariner', 5000, '2024-12-31', 'active');

2. SELECT with JOIN - View All Active Items (READ)

SELECT ai.item_name, s.name as seller_name, ai.starting_price, ai.current_bid, ai.end_date
FROM auction_items ai
JOIN sellers s ON ai.seller_id = s.seller_id
WHERE ai.status = 'active'
ORDER BY ai.end_date ASC;

3. UPDATE - Place New Bid (UPDATE)

UPDATE auction_items 
SET current_bid = 5500 
WHERE item_id = 1 AND 5500 > current_bid;

4. DELETE - Remove Bid (DELETE)

DELETE FROM bids 
WHERE bid_id = 5 AND status = 'retracted';

Normalization (3NF)

✅ First Normal Form (1NF): All attributes are atomic. Each cell contains a single value. No repeating groups.

✅ Second Normal Form (2NF): No partial dependencies. All non-key attributes depend on the entire primary key.

✅ Third Normal Form (3NF): No transitive dependencies. Seller details depend only on seller_id, item details only on item_id.

Constraints Implemented

• PRIMARY KEY: seller_id, item_id, bidder_id, bid_id
• FOREIGN KEY: auction_items.seller_id → sellers.seller_id
• FOREIGN KEY: bids.item_id → auction_items.item_id
• FOREIGN KEY: bids.bidder_id → bidders.bidder_id
• CHECK: starting_price > 0, current_bid >= starting_price
• CHECK: bid_amount > current_bid
• DEFAULT: auction_items.status = 'active'
• DEFAULT: bids.status = 'active'

Indexes for Performance

CREATE INDEX idx_item_status ON auction_items(status);
CREATE INDEX idx_item_end_date ON auction_items(end_date);
CREATE INDEX idx_bid_amount ON bids(bid_amount DESC);
CREATE INDEX idx_bid_time ON bids(bid_time);
